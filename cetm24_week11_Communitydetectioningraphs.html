Algorithm 1: Walktrap Community Detection¶

The first method used is the Walktrap algorithm, which finds communities by running many short random walks. The idea is that walks are more likely to stay within a dense community than to cross to another.

  • Finding Communities
    • wc <- cluster_walktrap(karate): This command runs the Walktrap algorithm on the karate graph. The results, which include the community structure, are stored in the wc object. The output shows this process identified 5 distinct communities.
  • Evaluating the result
    • modularity(wc): This calculates the modularity of the communities found by the Walktrap algorithm. Modularity is a score from -1 to 1 that measures how well a network is divided into communities. A higher score means denser connections within communities and sparser connections between them. The resulting modularity score is 0.353.
    • membership(wc): This command displays which community each node (person in the club) belongs to. For example, the output [1] 1 1 2 1 5... means the first node is in community 1, the second in community 1, the third in community 2, and so on.
  • Visualization
    • plot(wc, karate): This generates a plot of the karate club network, with nodes colored according to their community membership as determined by the Walktrap algorithm.
    • dendPlot(wc, mode="hclust", rect = 5): This command creates a dendrogram, which is a tree diagram that shows how the algorithm hierarchically merged nodes and smaller groups into the final communities. The rect = 5 argument draws rectangles around the five final clusters.

Algorithm 2: Louvain Community Detection¶

Next, the document applies the Louvain algorithm, a popular method that optimizes modularity to find communities.

  • Finding Communities
    • lc <- cluster_louvain(karate): This command executes the Louvain algorithm on the karate graph. The results are stored in the lc variable.
  • Evaluating the Result
    • modularity(lc): This calculates the modularity score for the Louvain partition, which is 0.419. This is higher than the Walktrap algorithm's score, suggesting a potentially better division of the network.
    • membership(lc): This displays the community assignments for each node. The output shows that this method found 4 communities.
    • sizes(lc): This command shows the number of members in each of the four communities: Community 1 has 12 members, Community 2 has 5, Community 3 has 11, and Community 4 has 6.
  • Visualization
    • plot(lc, karate): This plots the network again, but this time nodes are colored based on the 4 communities found by the Louvain algorithm.

Algorithm 3: Link Community Detection¶

Finally, the document demonstrates link community detection, which focuses on clustering the network's edges (links) rather than its nodes. This method is useful for identifying overlapping communities.

  • Preparation and Execution
    • library(linkcomm): This loads the linkcomm library needed for this specific algorithm.
    • karate_el <- get.edgelist(karate): The getLinkCommunities function requires the network to be in the form of an edge list (a list of connected pairs). This command converts the karate graph object into that format.
    • lc <- getLinkCommunities(karate_el, hcmethod = "single"): This runs the link community detection algorithm on the edge list. The hcmethod = "single" specifies the hierarchical clustering method to be used ("single-linkage").
  • Visualization and Analysis
    • plot(lc, type = "summary"): This plot provides a summary of the detected link communities, showing their size and how they relate to each other.
    • cr <- getClusterRelatedness(...): This calculates how similar the detected communities are to each other, again using a specified clustering method (ward.D).
    • cutDendrogramAt(...): This function takes the cluster relatedness dendrogram and "cuts" it at a specific height to form distinct meta-clusters.
      • Cutting at a height of 1.2 results in 2 communities.

# Function to merge small clusters like getLinkCommunities does
merge_small_clusters <- function(clusters, edge_list, min_size = 3) {
  cluster_sizes <- table(clusters)
  small_clusters <- as.numeric(names(cluster_sizes[cluster_sizes < min_size]))
  
  # Create new cluster assignments
  new_clusters <- clusters
  
  # Merge small clusters with their nearest larger cluster
  for(small_c in small_clusters) {
    edges_in_small <- which(clusters == small_c)
    if(length(edges_in_small) > 0) {
      # Find the most connected larger cluster
      # For simplicity, merge with the lowest numbered cluster
      new_clusters[edges_in_small] <- min(setdiff(unique(clusters), small_clusters))
    }
  }
  
  # Renumber clusters to be sequential
  unique_clusters <- unique(new_clusters)
  for(i in 1:length(unique_clusters)) {
    new_clusters[new_clusters == unique_clusters[i]] <- i
  }
  
  return(new_clusters)
}
